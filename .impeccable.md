## Design Context

### Users
Primary users are the project author and other data-inclined Teamfight Tactics players exploring new sets on day 0. They use the app as a fast local board-search tool to reason through possible late-game comps before the set has settled.

### Brand Personality
Information-dense, utilitarian, snappy. The interface should feel efficient and trustworthy, not ornamental or playful.

### Aesthetic Direction
Desktop-first, dark utilitarian workspace optimized for fast scanning and query refinement. The product should explicitly avoid border accents on rounded elements, glassmorphism, and gradient text.

### Design Principles
1. Make dense information scannable through hierarchy, not decoration.
2. Keep the shortest path to a new board search obvious and fast.
3. Prefer plain, domain-appropriate language over generic product copy.
4. Use feedback and system status to reinforce trust during long-running searches.
5. Reduce visual novelty when it slows interpretation or obscures data.
